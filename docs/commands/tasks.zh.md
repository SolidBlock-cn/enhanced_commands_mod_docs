# `/tasks`：管理服务器的任务

此[命令](...md)用于管理当前服务器的计划任务。

使用 `/setblocks`、 `/draw` 等命令修改大量方块，且没有设置 `immediately=true` 时，就会产生计划任务，从而将一个任务分到多个刻内完成，避免因一在个刻内完成大量任务而导致服务器卡顿。

## 语法

- `/tasks`：相当于 `/tasks list`。
- `/tasks count`：返回当前的计划任务的个数。
- `/tasks clear`：清除所有的计划任务，即使这些任务还没有执行完成。
- `/tasks remove <uuid>`：移除计划任务，即使这个任务还没有执行完成。
- `/tasks suspend <uuid>`：暂停某一个计划任务。对于已经暂停的任务会执行失败。
- `/tasks continue <uuid>`：继续执行一个已经暂停的计划任务。对于未暂停的任务会执行失败。
- `/tasks sprint <uuid> [进展数量]`：加速完成指定的计划任务，或者完成指定的步骤数量。这意味着即便有大量的进展未完成，也会在一个游戏刻内完成，可能会导致服务器卡顿一段时间。
- `/tasks list [限制]`：列举当前的计划任务，包括这些任务的名称、状态，并提供相应的按钮以进行操作。

## 参数

- `<uuid>`：任务的 UUID，如果不存在则会执行失败。可以先使用 `/tasks list` 获取当前的所有任务，然后再对特定的任务进行操作。
- `[进展数量]`：需要立即完成当前任务的步骤的数量。如果没有指定，则会立即将当前的任务执行完成。
- `[限制]`：限制当前页面展示的任务的个数。
