# `/postprocess` and `//postprocess`

This [command](...md) is used to postprocess blocks, such as fixing connection issues of fences and walls.

## Syntax

- `/postprocess <region> [keyword args]`
- `//postprocess [keyword args]`

This command is equivalent to `/etblocks <region> postprocess()` in effect, see [the `postprocess()` block function](/documents/arguments/block_function.md).

The command supports all keyword arguments of [`/setblocks`](../setblocks.md).
