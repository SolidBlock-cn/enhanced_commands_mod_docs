# `/undo` 和 `/redo`

此[命令](...md)用于撤销或还原此模组中的一些操作，只有部分模组支持此操作。

`/undo` 会撤销你的上一次操作，`/redo` 会撤销你的上一次撤销。如果你的上一次操作被设置为不可撤销，那么此命令会忽略被设置为不可撤销的操作，撤销上一次可撤销的操作。

`/undo` 产生的撤销操作可通过 `/redo` 撤销，由 `/redo` 撤销的撤销也可通过 `/undo` 去重新撤销。

实际上，每次可撤销的操作都会留下一次历史记录，因此 `/undo` 和 `/redo` 均是对历史记录进行撤销。由玩家执行的可撤销操作，其历史记录属于玩家。由命令方块、命令方块矿车、服务器等执行的可撤销操作，其历史记录属于服务器。如果没有可撤销的操作，也就是没有历史记录，那么命令将执行失败。关闭世界或关闭服务器后，撤销历史记录将不会保存。

举个例子，使用 `/setblocks sphere(5) ice` 在半径为 5 的球体范围内放置冰。此时如果执行 `/undo`，这些冰将消失，恢复为原来的方块。如果再执行 `/undo`，会失败，因为没有可撤销的操作了。如果在执行 `/undo` 后执行 `/redo`，那么这些冰会重新出现。

## 语法

- `/undo [关键字参数]`
- `/redo [关键字参数]`

## 参数

### `[关键字参数]`

本命令支持以下关键字参数。

#### `target`

实体选择器，只能选择一个玩家。默认为执行该命令的玩家（不受 `/execute as` 等命令修饰的影响）。撤销该玩家的操作。

在多人游戏内，可以用此参数可撤销他人的操作。

#### `target-server`

布尔值，默认为 `false`。如果设置为 `true`，则撤销的是服务器的操作。该参数不能和 `target` 参数同时存在。

由命令方块、命令方块矿车或服务器直接进行的操作，无法由玩家直接在聊天栏输入命令来撤销，但是如果添加 `target-server=true`，即可撤销这些操作。

#### `immediately`

布尔值，默认为 `false`。如果设置为 `true`，则即使被撤销的操作涉及大量方块，也将一次性完成，可能造成服务器卡顿一段时间。

需要注意的是，该参数与被撤销的操作无关。例如，如果使用 `/setblocks ... immediately=true` 一次性地填充了大量方块，那么使用此命令进行撤销时，如果没有指定 `immediately=false`，那么撤销过程仍是分步骤完成的。

#### `undoable`

布尔值，默认为 `false`。如果设置为 `true`，对于 `/undo`，该次撤销操作将无法通过 `/redo` 撤销。对于 `/redo`，该次撤销无法通过 `/undo` 撤销。

## 示例

- `/undo`：撤销最近的一次可撤销的操作。
- `/redo`：将最近的一次可撤销的撤销操作撤销。
- `/undo target=Steve`：撤销玩家 Steve 最近的一次可撤销的操作。
- `/undo target-server=true`：撤销服务器最近的一次可撤销的操作。
- `/undo target=Steve target-server=true`：无效命令，因为 `target` 和 `target-server` 参数不同同时存在。
- `/undo immediately=true`：立即完成一次撤销操作，即使该操作可能涉及大量方块更改。
- `/undo undoable=false`：撤销最近的一次可撤销操作，并且该撤销操作不可使用 `/redo` 撤销。
- `/redo undoable=false`：撤销最近的一次可撤销的撤销操作，并且该撤销操作不可使用 `/undo` 撤销。
