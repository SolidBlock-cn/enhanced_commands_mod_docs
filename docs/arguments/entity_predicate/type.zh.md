# `type`：测试实体的类型

此[实体谓词](index.md)测试实体的类型。可以一次指定一个类型，也可指定多个类型。指定的类型可以是实体类型 id，也可以是实体类型的标签。此参数也可以取反。

通常来说，对于实体选择器而言，如果 `type` 参数只指定一个类型、不是标签且没有取反的话，会直接用作实体选择器自身的属性，并有可能直接影响到实体选择器本身选择实体的逻辑。

## 语法

- `[type=<类型>]`：选择指定类型的实体。这里的 `<类型>` 可以是实体 id（如 `cat`），也可能是标签（如 `#skeletons`），下同。
- `[type=!<类型>]`：选择不是指定类型的实体。
- `[type=<类型 1>|<类型 2> ...]`：选择属于多个类型中的任意一个类型的实体。这里的类型不能重复，下同。
- `[type=!<类型 1>|<类型 2> ...]`：选择不属于多个类型中的任何类型的实体。

## 示例

- `@p[type=cat|wolf]`：距离最近的猫或狼。
- `@e[type=player|iron_golem]`：所有的存活的玩家或者铁傀儡。
- `@e[type=zombie|zombie]`：无效，因为有重复的值。

## 数据结构

若只指定一个类型，且不是标签：

- `type`：此处为 `"enhanced_commands:type"`。
- `entity_type`：字符串。
- `inverted`：布尔值，可选，默认为 `false`。

若只指定一个类型，且是标签：

- `type`：此处为 `enhanced_commands:type_tag"`。
- `tag`：字符串，标签 id，带有井号。
- `inverted`：布尔值，可选，默认为 `false`。

若指定了多个类型：

- `type`：此处为 `enhanced_commands:types`。
- `types`：列表。
    - 列表的元素。字符串。实体 id 或标签 id（带有井号）。
- `inverted`：布尔值，可选，默认为 `false`。
