# `gamemode`：测试玩家的游戏模式

此[实体谓词](index.md)用于测试玩家的游戏模式。和原版的一样，如果实体选择器使用了此谓词，那么整个实体选择器将会只选择玩家。

## 语法

- `[gamemode=<游戏模式>]`：选择指定游戏模式的玩家。
- `[gamemode=!<游戏模式>]`：选择不属于指定游戏模式的玩家。
- `[gamemode=<游戏模式 1>|<游戏模式 2>...]`：选择属于指定游戏模式中任一游戏模式的玩家。
- `[gamemode=!<游戏模式 1>|<游戏模式 2>...]`：选择不属于指定游戏模式中任何游戏模式的玩家。

游戏模式可以使用全称（如 `creative`）、简写（如 `c`）和数字（如 `1`）。可以指定多个游戏模式，但是不能有重复（即使是写成不同形式的）。

参数名称 `gamemode` 可以简写为 `m`，二者等价。

> 注意：此谓词在命令中的实体选择器对应的参数名称是 `gamemode`，没有下划线，但其内部实体谓词类型 id 是 `enhanced_commands:game_mode`，有下划线。

## 示例

- `[gamemode=a]`：选择冒险模式的玩家。
- `[gamemode=!c]`：选择不是冒险模式的玩家。
- `@a[m=c|a]`：选择创造模式或者冒险模式的玩家。
- `@a[m=!c|s]`：选择既不是创造模式又不是生存模式的玩家。
- `@a[m=c|1]`：无效，因为 `c` 和 `1` 是重复的。
- `@e[m=!spectator|c]`：选择既不是旁观模式又不是创造模式的存活玩家。尽管使用的是 `@e`，但此选择器只选择玩家。
- `@a[m=!spectator|3]`：无效，因为 `spectator` 和 `3` 是重复的。

## 数据结构

- `type`：此时为 `"enhanced_commands:game_mode"`。
- `game_mode`：字符串或列表。游戏模式的名称。
    - 字符串，游戏模式的名称（若 `game_mode` 为列表）。
- `inverted`：布尔值，可选，默认为 `false`。
