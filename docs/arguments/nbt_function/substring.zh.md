# `substring()` 函数：截取子字符串

此 [NBT 函数](index.md)用于截取子字符串。如果参数不是字符串，会抛出错误。

## 语法

- `substring(<开始索引>, [结束索引])`
- `substring(<开始索引>, [结束索引], [关键字参数：lenient | original])`

## 参数

### `<开始索引>`

整数。表示从字符串的第几位开始截取。支持负索引。

### `[结束索引]`

整数。表示截取到字符串的第几位前。支持负索引。

如果未指定，则截取到字符串末尾。

### 关键字参数

#### `lenient`

布尔值，默认为 false。

- 若为 false，当开始索引或结束索引无效（大于字符串实际长度，或小于字符串长度的相反数），或者开始索引的实际索引大于结束索引的实际索引时，抛出异常。若被截取子字符串的 NBT 不是字符串，也会抛出异常。
- 若为 true，则上述几种情况下都不会抛出异常，而是直接返回原来的值，除非被截取的 NBT 为 null。

#### `original`

NBT 函数，可选。表示原始值。

- 若未指定，则被截取字符串的 NBT 为该 NBT 函数运行时的参数。
- 若指定，则被截取字符串的 NBT 为该参数的值。该参数是 NBT 函数，会先运行此 NBT 函数，再对该 NBT 函数的结果进行替换。

## 示例

- `substring(2, 4)`：截取第 3 到 4 位的子字符串。
    - 应用于 `"apple"`：返回 `"pl"`。
    - 应用于 `"sandwich"`：返回 `"nd"`。
    - 应用于 `"mc"`：错误。
    - 应用于 `1234`：错误。
    - 应用于 `["a", "b"]`：错误。
- `substring(2)`：从第 3 位截取到末尾。
    - 应用于 `"apple"`：返回 `"ple"`。
    - 应用于 `"sandwich"`：返回 `"ndwich"`，
    - 应用于 `"mc"`：返回空字符串。
- `substring(3, -3)`：从第 4 位截取到倒数第 4 位。
    - 应用于 `"apple"`：错误。
    - 应用于 `"sandwich"`：返回 `"dw"`。
    - 应用于 `"mc"`：错误。
- `substring(5, 3)`：应用于任何字符串都是错误。
- `substring(-5, -3)`：从倒数第 5 位截取到倒数第 4 位。
    - 应用于 `"apple"`：返回 `"ap"`。
    - 应用于 `"sandwich"`：返回 `"dw"`。
    - 应用于 `"mc"`：错误。
- `substring(-3, -5)`：应用于任何字符串都是错误。
- `substring(1, 3, original = "cake")`：对于任何 NBT，都返回 `"ak"`。
- `substring(2, 4, lenient = true)`：
    - 应用于 `"apple"`：返回 `"pl"`。
    - 应用于 `"sandwich"`：返回 `"nd"`。
    - 应用于 `"mc"`：返回 `"mc"`（不变）。
    - 应用于 `1234`：返回 `1234`（不变）。
    - 应用于 `["a", "b"]`：返回 `["a", "b"]`（不变）。

## 数据格式

- **start_index**：整数。
- **end_index**：整数，可选。
- **lenient**：布尔值，可选，默认为 false。
- **original**：复合标签，表示 NBT 函数，可选。
