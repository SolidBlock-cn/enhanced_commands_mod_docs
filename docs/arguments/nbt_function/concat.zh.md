# `concat()` 函数：连接多个字符串

此 [NBT 函数](index.md)可用于连接多个字符串。可以连接字符串与数字，数字会转化为字符串（不带数据类型后缀）。连接过程中如果遇到其他类型的值，则会报错。

## 语法

- 语法 1：`concat(<NBT 函数 1>, <NBT 函数 2> ...; delimiter = [分隔符])`：直接连接多个 NBT 函数所返回的结果。
- 语法 2：`concat(* <NBT 函数>; delimiter = [分隔符])`：如果这个 NBT 函数应用的结果是列表，则会连接列表的各个元素；如果不是列表，则会报错。

## 参数

### `<NBT 函数> ...`（语法1）

需要被连接的 [NBT 函数](index.md)。可以不限数量。在语法 1 中，每个 NBT 函数在应用时都直接将以整个函数的参数作为参数。如果有一个 NBT 函数返回的值不是字符串或数字，则会报错。

### `<NBT 函数>`（语法2）

需要被作为列表连接的 [NBT 函数](index.md)。如果该 NBT 函数应用的结果不是列表，则会报错。如果是列表，则会连接该列表中的各元素。如果这些元素有一个不是字符串或数字，则会报错。

### `delimiter = [分隔符]`

[NBT 函数](index.md)。其应用结果的字符串形式将用作连接各元素时的分隔符。如果这个函数返回的结果不是字符串或数字，则会报错。

## 示例

### 语法 1 示例

- `concat(a, b, c)`：返回字符串 `"abc"`。
- `concat(a, 1, 5)`：返回字符串 `a15`。
- `concat(1L, 2s)`：返回字符串 `12`。连接时不带数据类型后缀。
- `concat(true, false)`：返回字符串 `10`，因为 `true` 和 `false` 实际上是 `1b` 和 `0b`。
- `concat(1, [], 2)`：无法执行，因为列表无法用于连接。
- `concat(1, {}, a)`：无法执行，因为复合标签无法用于连接。

### 语法 2 示例

- `concat(* 1)`：无法执行，因为 1 不是列表。
- `concat(* {})`：无法执行，因为 `{}` 不是列表。
- `concat(* [1, 2, 3])`：返回 `123`。
- `concat(* [a, 1, 5])`：返回 `a`，因为 `[a, 1, 5]` 这个 NBT 列表函数在应用时，后面的 1 和 5 两个元素因类型不一致，无法被实际添加到列表，因此该函数只是在 `[a]` 这个列表中进行连接。

## 数据结构

* **type**：字符串。当前为 `enhanced_commands:concat`。
* **flatten**：布尔值。该 NBT 函数是否为展平模式。
* **elements**：列表。仅限 `flatten: false` 时，各 NBT 函数。
    * 复合标签。一个 NBT 函数。
* **element**：复合标签。仅限 `flatten: true` 时，一个 NBT 函数。
* **delimiter**：复合标签。一个 NBT 函数。
