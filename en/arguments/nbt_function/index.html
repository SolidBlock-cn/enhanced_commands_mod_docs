    NBT 函数¶

    NBT 函数（NBT function）可用于修改 NBT 的数据，使用时会应用在已有的一个 NBT 元素上（可以是任何类型，且可以为空，这个元素可称为应用时的“参数”），然后返回一个 NBT 元素（非空）。NBT 函数的大多数语法与 NBT 的语法非常类似，同时支持一些特殊的语法。

    使用 /testarg nbt_function <NBT 函数> apply [NBT 元素] 可测试 NBT 函数并查看应用该 NBT 函数后的结果。在该命令中，关键字 apply 后的 NBT 元素将作为该 NBT 函数应用时的参数。

    简单的 NBT 函数¶

    简单值¶

    NBT 函数可以返回任意指定的 NBT 元素，和 NBT 元素自身的写法基本相同。这些 NBT 函数会无视参数而直接返回这些指定的 NBT 元素。

    • 3b：返回字节值 3b。
    • 4s：返回短整型 4s。
    • "str"：返回字符串 "str"。

    Tip

    可以用 /testarg 命令来测试 NBT 函数，例如运行 /testarg nbt_function "str" apply {key: value} 可以得到 "str"。 后面的这些例子都可以使用同样的方法来测试。

    等号值¶

    等号值是指当参数是任意一种 NBT 的数字类型时，修改其数值但是保持其数据类型不变，如果参数不是数字类型，则忽略该参数，返回指定的值：

    • =3：若参数为数字，则返回该数据类型的 3，否则返回整数 3。
    • =3b：若参数为数字，则返回该数据类型的 3，否则返回字节值 3b。
    NBT 函数 参数 返回结果
    =3 2s 3s
    =3 2b 3b
    =3 "str" 3
    =3b 2s 3s
    =3b 2b 3b
    =3b "str" 3b

    示例

    使用 /testarg 命令来测试 NBT 函数的例子：运行 /testarg nbt_function =3b apply 2 可以得到 3。

    复合标签¶

    NBT 复合标签函数用于操作复合标签，如果参数不是复合标签，会应用于空复合标签，再对空复合标签进行操作并返回。也就是说，无论参数是什么类型，NBT 复合标签函数始终返回复合标签。

    合并复合标签¶

    直接使用类似于复合标签语法，可表示一个合并复合标签的 NBT 函数，该 NBT 函数用于修改复合标签。当参数不是复合标签时，会返回对空复合标签应用的结果。我们知道，复合标签是 NBT 的一种数据类型，其在 SNBT 中会用 {} 括起来，里面可以有多个字段，每个字段都有键和值。例如，NBT 复合标签：{item: {id: stone}} 有一个字段，这个字段的键为 key，值为 NBT 复合标签 {id: stone}。

    复合标签 NBT 函数内支持以下语法：

    • <键>: <NBT 函数> 或 <键> = <NBT 函数>：对参数中的指定键对应的值应用指定的 NBT 函数，如果参数中不存在这个键，则会新增字段，其值为 NBT 函数对 null 应用的结果。
    • - <键>：从复合标签中移除指定的键的字段。注意“-”和键之间必须要有空格，否则会连同连字符一起视为普通的键。

    例如：

    • {a: b, c: d}：将原先的复合标签中的 a 的值替换为字符串 b，c 的值替换为字符串 d。例如，如果参数为 ({a: "value", e: f})，那么该 NBT 函数返回 {a: b, c: d, e: f}，其中 a 字段被函数中的值替换，c 字段从函数中的值中加入，e 字段未在函数中定义因此未被影响。
    • {Color: 1b}：对复合标签的 Color 字段的值 2 应用 1b 函数。例如，对于 {Color: 2} 返回 {Color: 1b}。
    • {Color = 1b}：对复合标签的 Color 字段的值 2 应用 =1b 函数。例如，对于 {Color: 2} 返回 {Color: 1}。
    • {- a}：从复合标签中移除 a。例如，{a: b, c: d} 返回 {c: d}。
    • {-a: b}：由于没有空格，-a 被视为常规的键，该函数会将原先的复合标签中的 -a 字段的值设为字符串 b，例如 {a: b, c: d} 返回 {a: b, c: d, -a: b}。
    • {-a}：无效的 NBT 函数。
    NBT 函数 参数 返回结果
    {a: b} {} {a: b}
    {a: b} [1, 2, 3] {a: b}
    {a: b} {a: 123} {a: b}
    {a: b} {a: 123, c: d} {a: b, c: d}
    {a: {c: d}} {} {a: {c: d}}
    {a: {c: d}} {a: 123} {a: {c: d}}
    {a: {c: d}} {a: {e: f}} {a: {c: d, e: f}}
    {- a} {a: b} {}
    {- a} {a: b, c: d} {c: d}

    替换复合标签¶

    在复合标签前添加等号，可完全替换整个 NBT 复合标签，相当于对空的 NBT 复合标签应用 NBT 函数。

    NBT 函数 参数 返回结果
    ={a: b} {} {a: b}
    ={a: b} [1, 2, 3] {a: b}
    ={a: b} {a: 123} {a: b}
    ={a: b} {a: 123, c: d} {a: b}
    {a = {c: d}} {a: {e: f}, g: h} {a: {c: d}, g: h}
    = {a: {c: d}} {a: {e: f}} {a: {c: d}}

    当参数不是 NBT 复合标签时，无论是否带等号，效果均相同。

    列表¶

    NBT 列表函数用于操作列表，如果参数不是列表，会应用于空列表，再对空列表进行操作并返回。也就是说，无论参数是什么类型，NBT 列表函数始终返回列表。

    在 1.21.5 之前，NBT 列表不支持混入多种不同类型元素，但此语法不会检查元素类型。尝试混入多种不同类型元素时，相应的操作会被忽略，但也不会输出任何错误。

    替换列表元素¶

    一般情况下，如果在 NBT 函数中使用列表的语法，会创建一个替换列表的 NBT 函数。无论参数为何值，都会被替换为一个新的列表。在 NBT 函数语法中，列表的各个元素都是 NBT 函数，在应用时，这些 NBT 函数均会以 null 为参数应用，作为返回的列表中的元素。

    • [a, b, c]：无论参数是什么，返回一个列表，有 3 个元素。
    • []：无论参数是什么，返回空的列表。
    NBT 函数 参数 返回结果
    [a, b, c] [1]
    [4,5,6]
    []
    123    		 [a, b, c]
    [] [1]
    [4,5,6]
    []
    123    		 []

    在 1.21.5 之前的版本中，NBT 函数 [123, abc, true] 是有效的，但是应用后只会返回 [123]。

    修改指定索引的列表元素¶

    可以在值前指定索引，表示替换特定位置的值，特定位置的值不存在时会被忽略。

    • [0: A]：将列表的第 1 个值替换为 A。例如，[a, b, c] 返回 [A, b, c]。
    • [1: A]：将列表的第 2 个值替换为 A。例如，[a, b, c] 返回 [a, A, c]。
    • [-1: A]：将列表的最后一个值替换为 A。例如，[a, b, c] 返回 [a, b, A]。
    • [3: A]：将列表的第 4 个值替换为 A。例如，对于 [a, b, c]，由于列表没有第 4 个值，返回 [a, b, c]。
    NBT 函数 参数 返回结果
    [0: A] [a, b, c] [A, b, c]
    [1: A] [a, b, c] [a, A, c]
    [-1: A] [a, b, c] [a, b, A]
    [3: A] [a, b, c] [a, b, c]

    此外，也可以既指定替换列表，又设置特定索引的值。

    在 NBT 函数语法中，如果列表非空且各元素均带有索引，那么不会替换整个列表内容，只修改指定索引。如果列表为空，或者有任何不带索引的元素，那么会先替换整个列表内容。

    • [d, e, f, 1: G]：将列表内容替换为 [d, e, f]，再将第 2 个元素替换为 G。
    NBT 函数 参数 返回结果
    [d, e, f, 1: G] [a, b, c] [d, G, f]
    [d, e, f, 2: G] [a, b, c] [d, e, G]
    [d, e, f, 3: G] [a, b, c] [d, e, f]

    在列表中插入值¶

    如果列表中有省略号，那么这个列表的元素均会被插入到被应用的 NBT 列表中（实际上是对 null 应用 NBT 函数）。其中，在 NBT 函数语法中，省略号前面的不带索引的元素会被插入到列表最前面，省略号后面的不带索引的元素会被插入到列表的最后面，带索引的元素无论是在省略号前面还是后面均会被插入到指定的索引位置。如果指定的索引不存在，则不会操作，也不会输出错误。

    一个 NBT 列表语法中，省略号只能出现一次。

    • [A, ...]：在整个列表前插入字符串 A。例如，对于 [a, b, c] 返回 [A, a, b, c]。
    • [A, B, ...]：在整个列表前插入字符串 A 和 B。例如，对于 [a, b, c] 返回 [A, B, a, b, c]。
    • [..., A]：在整个列表后插入字符串 A。例如，对于 [a, b, c] 返回 [a, b, c, A]。
    • [A, ..., B]：在整个列表前插入字符串 A，在列表后插入字符串 B。例如，对于 [a, b, c] 返回 [A, a, b, c, B]。
    • [A, ..., B, ..., C]：无效，因为省略号出现了两次。
    • [2: A, ...]：将字符串 A 插入到第 3 个元素的位置。例如，对于 [a, b, c] 返回 [a, b, A, c]。
    • [..., 2: A]：和前一个例子等价，
    • [-2: A, ...]：将字符串 A 插入到倒数第 2 个元素的位置。例如，对于 [a, b, c] 返回 [a, b, A, c]。
    • [..., -2: A]：和前一个例子等价。
    NBT 函数 参数 返回结果
    [A, ...] [a, b, c] [A, a, b, c]
    [A, B, ...] [a, b, c] [A, B, a, b, c]
    [..., A] [a, b, c] [a, b, c, A]
    [A, ..., B] [a, b, c] [A, a, b, c, B]

    Note

    此类 NBT 函数的应用顺序为：1. 插入到列表之前（不带索引），2. 插入到列表之后（不带索引），3. 插入到指定索引的位置（按其出现的顺序）。例如，NBT 函数 [a, 5: b, ..., 3: c, d] 会先将 a 插入到最前面，再将 d 插入到最后面，再将 b 插入到第 6 个元素位置，再将 c 插入到第 4 个元素位置。

    需要注意的是，当你执行一次插入操作后，元素的索引会发生改变。例如，对 [a, b, c] 应用 NBT 函数 [..., 2: d, 2: e] 会返回 [a, b, e, d, c]，因为会先将 d 插入到第 3 个元素位置，变成 [a, b, d, c]，再将 e 插入到第 3 个元素位置，变成 [a, b, e, d, c]。

    NBT 函数的复合语法¶

    使用 | 连接多个 NBT 函数，表示选取一个随机的 NBT 函数；例如，a|b|c 会返回 a 或 b 或 c。

    Tip

    可以尝试运行多次 /testarg nbt_function a|b|c apply。

    使用 * 连接多个 NBT 函数，表示依次应用多个 NBT 函数。例如，NBT 函数 lime_wool*string_replace(lime, blue)*string_replace(wool, concrete) 对任何参数都会返回 blue_concrete（先将参数替换为字符串 lime_wool，再对该字符串运行 NBT 函数 string_replace(lime, blue) 并返回 blue_wool，再对该字符串应用 NBT 函数 string_replace(wool, concrete) 返回 blue_concrete）。

    上面的这些语法不能有空格，但是如果位于括号内，则可以有空格。参见 block_function。

    NBT 函数的函数语法¶

    NBT 函数支持以下函数语法：

    • concat()：连接多个 NBT 字符串。
    • from()：从另一个 NBT 来源导入数据。
    • overlay()：依次运行多个 NBT 函数。
    • pick()：随机选择一个 NBT 函数。
    • pos()：将坐标以双精度浮点数数组的形式返回。
    • regex_replace()：进行正则表达式文本替换。
    • replace()：替换 NBT 中所有符合指定的 NBT 谓词的值，并应用 NBT 函数。
    • string_replace()：进行字符串替换。
    • substring()：截取子字符串。

    数据类型¶

    • type：带命名空间的 ID，NBT 函数的类型，若未指定命名空间，则以 enhanced_commands 为命名空间，以下均省略命名空间（例如下文的 compound 表示 enhanced_commands:compound）。本页仅列出介绍于本页的类型，其他的类型请见各个子页面。
    • （当 type 为 "compound"：）
    • source：复合标签。
      • 键为对应 NBT 中的键，值为复合标签。
        • value：NBT 函数，可选。如果不存在，则表示从 NBT 中移除值。
    • allows_merge：字符串。
    • （当 type 为 "list_insertion"：）
    • insert_before：列表。
      • NBT 函数。
    • insert_after：列表。
      • NBT 列表。
    • insert_positional：列表。
      • 复合标签。
        • index：整数。
        • value：NBT 函数。
    • （当 type 为 "list_ops"：）
    • value_replacements：列表。
      • NBT 函数。
    • positional_functions：列表。
      • 复合标签。
        • index：整数。
        • value：NBT 函数。
    • （当 type 为 "number_value"：）
    • value：NBT 数字。
    • （当 type 为 "simple"：）
    • value：NBT 元素。
    • （type 可能为其他的值，请参见各子页面，其中 NBT 函数的复合语法实际上等价于 overlay() 和 pick()。）
